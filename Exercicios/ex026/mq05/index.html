Por que mobile first faz sentido hoje?

Mais de 60% do tráfego da internet vem de dispositivos móveis. Isso significa que a maioria das pessoas que vão acessar seu projeto provavelmente estão no celular. Começar pelo mobile não é só uma boa prática técnica — é uma decisão estratégica que coloca o usuário real no centro do desenvolvimento.

Como isso muda sua forma de pensar o CSS?

Quando você escreve CSS mobile first, seus estilos base são simples e diretos: uma coluna, fontes legíveis, botões com área de toque adequada. À medida que a tela cresce, você vai adicionando complexidade com min-width. Isso cria um código mais limpo e previsível, porque você está construindo de baixo para cima — do simples para o complexo — em vez de ficar "consertando" um layout grande para caber em telas pequenas.

O impacto na performance

Dispositivos móveis geralmente têm conexões mais lentas e menos poder de processamento. Ao pensar mobile first, você naturalmente evita carregar recursos pesados desnecessários — imagens gigantes, animações complexas, fontes demais. O resultado é um site mais rápido para todo mundo, não só para mobile.

Na prática, para um projeto simples

Imagine um portfólio básico. Mobile first significa: primeiro você faz os cards de projeto empilhados em coluna, o menu vira um hambúrguer, e o texto tem tamanho confortável para leitura no polegar. Só depois você abre o layout em duas ou três colunas para desktop com um @media (min-width: 768px). Essa ordem de raciocínio te poupa retrabalho e torna o código muito mais fácil de manter.

Um erro comum de quem está aprendendo

Muitos devs iniciantes escrevem todo o CSS pensando no desktop e depois tentam "espremer" tudo para mobile com max-width. Isso gera gambiarras, código duplicado e layouts quebrados. Quando você inverte essa lógica e começa pelo menor, percebe que a maioria dos problemas de responsividade simplesmente não aparecem.
